ViewChild et ViewChildren sont l'équivalent, ViewChild est pour un élément, alors que ViewChildren est pour plusieurs éléments. Ils servent à maintenir des références avec des objects dans nos templates (HTML). À de ses références, on pourra faire s'abonner à des évènements (.subscribe) ou encore les utiliser directement pour instancier une librairie JS. Pour les utiliser, on doit mettre #nomDeLaReference dans la balise HTML 
      {{codeHTMLChild}}
Ensuite, nous pourront utiliser cette référence pour créer un ViewChild dans le .ts 
      {{codeTSChild}}
Le fonctionnement pour un ViewChildren est relativement le même, mais il s'applique à plusieurs éléments On doit mettre #nomDeLaReference dans la balise HTML qui contient un ngFor 
      {{codeHTMLChildren}}
Ensuite, nous pourront utiliser cette référence pour créer un ViewChild dans le .ts 
      {{codeTSChildren}}
Finalement, nous pourrons nous abonner aux changements dans le AfterViewInit. 
      {{codeSubscribeChildren}}
